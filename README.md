# CSLangQuestProjects

Computer Science 언어 학습과 실전 프로젝트를 단계적으로 기록하는 저장소입니다. 저수준 개념(메모리, 동시성)부터 서비스/스마트컨트랙트까지 언어별 실습을 한 흐름으로 묶어, 배운 내용을 바로 코드로 검증하는 것을 목표로 합니다.

## 배경
- 여러 언어를 점진적으로 익히되, 각 언어에서 “왜 이렇게 설계됐는가?”를 비교·회고하며 이해를 깊게 하기 위함입니다.
- 모든 학습 산출물(코드, 실행 예시, 회고)을 한 저장소에 모아 재사용성과 검색성을 높입니다.
- 짧은 실습 → 미니 프로젝트 → 회고로 이어지는 반복 루프로 학습 효율을 극대화합니다.

## 목표
- CS 기초(자료구조, 알고리즘, 시스템)에 필요한 언어 실습을 꾸준히 진행
- 짧은 주기의 실전 예제를 통해 개념을 빠르게 검증
- 프로젝트별 회고를 남겨 학습 루프를 완성

## 구조 제안
- `<언어>/basics/`: 문법, 자료구조/알고리즘 기초 예제
- `<언어>/exercises/`: 짧은 문제 풀이 및 실습 스니펫
- `<언어>/projects/`: 주제별 미니 프로젝트(README 포함)
- `<언어>/notes/`: 요약 정리, 회고 및 참고 링크

## 언어 학습 순서 제안
- Rust → Go → Solidity
  - Rust: 안전한 시스템 프로그래밍 감각과 소유권 모델 체득
  - Go: 동시성·서비스 개발을 단순 문법으로 빠르게 익힘
  - Solidity: 스마트 컨트랙트는 보안/가스 모델 특화, 마지막에 집중 학습

### 진행 팁
- `<언어>` 루트에서 `basics`→`exercises`→`projects` 순으로 난이도 상승
- 최소 단위 실습마다 `<언어>/notes/`에 회고를 남기고, 다음 목표를 명시
- 실습 단위는 30~90분 내로 작게 쪼개고, 끝날 때마다 실행/테스트 결과를 README나 노트에 기록

## 시작 가이드
1) 이 저장소를 로컬에 클론합니다.
2) 집중할 언어의 `/<언어>/basics/`부터 작은 단위 예제를 작성하고, 실행 방법/출력/배운 점을 README에 기록합니다.
3) 간단한 문제풀이/실험은 `/<언어>/exercises/`에 두고, 재현 가능한 입력·출력 예시를 주석 또는 README에 남깁니다.
4) 새 프로젝트를 시작할 때는 `<언어>/projects/<주제>` 폴더를 만들고 목표/기능/사용법/회고를 기록합니다.
5) 마무리 후 `<언어>/notes/`에 회고(무엇을 배웠는지, 막힌 점/해결, 다음 할 일)를 추가합니다.

## 추천 워크플로우 (예시)
- 스프린트 길이: 1~2주
- 데일리: 1~2개 실습(30~90분) + 짧은 회고
- 주간: 미니 프로젝트 1개, README에 문제 정의/설계/테스트/회고 정리

## 브랜치 & 커밋 가이드
- 작은 단위 기능/예제마다 브랜치를 만들어 작업 후 병합
- 커밋 메시지 예시 (유형별)
  - 기능 추가: `feat: add rust ownership notes`, `feat: go goroutine demo`
  - 버그 수정: `fix: go channel leak in worker`, `fix: rust borrow example typo`
  - 문서/정리: `docs: add solidity gas tips`, `docs: update roadmap`
  - 환경/유지보수: `chore: bump toolchain versions`, `chore: add makefile lint target`
